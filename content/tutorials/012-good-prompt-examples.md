## 同一个任务，为什么两种问法结果差很多？

同样是让 AI 写一段客户跟进消息，两种问法得到的结果会差很多。

第一种问法很常见：

帮我写一段客户跟进话术。

第二种问法会多说一些信息：

我是做企业服务销售的。这个客户上周了解过我们的方案，但还没有确认是否继续沟通。请帮我写一段微信跟进消息，语气自然、克制，不要催得太急，目标是约一个 15 分钟电话沟通。请给我 3 个版本，并说明每个版本适合什么场景。

两种问法都不复杂，但 AI 能拿到的信息完全不同。

第一种问法，AI 只能猜。它不知道你是谁，不知道客户现在处在哪个阶段，也不知道这段话是发在微信里，还是写在正式邮件里。于是结果很容易变成一段标准、客气、但不太像真实沟通的话。

第二种问法，AI 至少知道：你是谁、客户状态、使用场景、语气限制、最终目标和输出方式。

这就是高质量 Prompt 的起点。

不是越长越好，而是该说的信息说清楚。

:::tutorial-compare-table
title: 模糊 Prompt vs 高质量 Prompt
帮我写一段客户跟进话术|说明身份、客户状态、使用场景、语气限制、目标和输出格式
让 AI 自己猜关系和语气|提前告诉 AI 当前阶段和不要出现的感觉
只要一段话术|要求 3 个版本，并说明每个版本适合什么场景
:::

## 好 Prompt 不是越长越好，而是信息够清楚

有些人看到别人写的 Prompt 很长，就以为越长越好。

其实不是。

一条 Prompt 如果堆了很多身份设定、专业词、夸张要求，但没有说清真实任务，照样不好用。

好 Prompt 的重点不是字数，而是减少 AI 的猜测。

如果一条 Prompt 没有说清你是谁、要完成什么、当前背景是什么、有哪些限制、最后要什么格式，AI 就只能按常见套路回答。它可能写得很顺，但不一定适合你的真实场景。

反过来，一条 Prompt 不一定很长。只要把关键情况说清楚，结果就会稳定很多。

这一步不用想复杂。你可以先问自己一句：如果我把这个任务交给一个真实同事，他需要知道哪些信息，才能不乱猜？

这些信息，通常也应该告诉 AI。

## 拆一个高质量 Prompt，看它到底好在哪里

先把这条客户跟进 Prompt 拆开看。

完整版本是：

我是做企业服务销售的。这个客户上周了解过我们的方案，但还没有确认是否继续沟通。请帮我写一段微信跟进消息，语气自然、克制，不要催得太急，目标是约一个 15 分钟电话沟通。请给我 3 个版本，并说明每个版本适合什么场景。

它不是因为写得长才有效，而是每一段都在帮 AI 减少猜测。

:::tutorial-flow
title: 案例拆解表
我是做企业服务销售的|说明身份和行业；可以替换成课程顾问、社群运营、老师、小店老板、自由职业者。
客户上周了解过我们的方案|说明客户当前阶段；可以替换成刚咨询、已经试用、还在比较、之前拒绝过。
微信跟进消息|说明使用场景；可以替换成邮件、社群私信、电话前短信、朋友圈私聊。
语气自然、克制，不要催得太急|说明语气限制；可以替换成正式、亲和、简洁、不要像广告、不要太强势。
目标是约一个 15 分钟电话沟通|说明真实目标；可以替换成确认需求、约时间、收集反馈、提醒提交资料。
给我 3 个版本，并说明每个版本适合什么场景|说明输出格式；可以替换成表格、清单、多个版本、优先级排序、步骤说明。
:::

你以后看到别人写得不错的 Prompt，不要只看它用了什么高级词。

更应该看：它有没有把任务里最容易被 AI 猜错的地方提前说清楚。

## 普通人写 Prompt，先补这 5 类信息

普通人写 Prompt，可以先补 5 类信息。

:::tutorial-flow
title: 高质量 Prompt 5 类信息
身份|我是谁，或者希望 AI 扮演什么角色。例如：我是企业服务销售，你是一名擅长客户沟通的顾问。
任务|这次到底要完成什么。例如：写一段客户跟进微信。
背景|当前发生了什么，为什么要做这件事。例如：客户上周了解过方案，但还没有确认下一步。
限制|哪些不能做，哪些语气不要出现。例如：不要催太急，不要夸大承诺，不要像广告。
输出格式|最终希望 AI 怎么交付。例如：给 3 个版本，并说明每个版本适合什么场景。
:::

身份让 AI 知道应该站在哪个位置回答。任务让 AI 不要把问题扩展成一堆无关建议。背景让结果贴近真实关系和进度。限制能减少过度包装。输出格式则让结果更容易比较和选择。

你不需要每次都写很长。

但这 5 类信息里，缺得越多，AI 越容易走向通用回答。

## 复制案例前，先改成自己的场景

看到别人的好 Prompt，不要急着整段复制。

因为别人的行业、对象、关系和目标，通常和你不一样。你做销售，他做运营；你要发给客户，他要发给社群成员；你希望克制一点，他可能希望更有行动感。

复制前，至少先改 5 处。

:::tutorial-flow
title: 复制前先改这 5 处
行业|把企业服务销售改成你的职业或身份。
对象|把客户改成你的真实沟通对象，比如老板、同事、学生、用户、社群成员。
当前状态|说明对方现在处在哪一步，比如刚咨询、已经试用、还在犹豫、已经拒绝过。
语气|说明你希望自然、正式、亲和、克制，还是更有行动感。
输出格式|说明你要一个版本、三个版本、表格、清单，还是步骤说明。
:::

这 5 处改完以后，别人的 Prompt 才开始变成你的 Prompt。

如果不改，AI 可能会给你一段看起来很完整、但关系错位、语气不对、目标也不准确的内容。

复制别人的提示词，不等于学会了提示词。关键不是背模板，而是看懂结构。

## 哪些结果不能只看顺不顺

高质量 Prompt 可以提高结果稳定性，但不能保证结果一定正确、一定适合发布、一定适合真实关系。

你仍然需要判断：背景是否真实，语气是否适合对象，是否有过度承诺，是否遗漏关键信息，是否适合直接发送或发布。

:::tutorial-flow
title: Prompt 结果判断卡
背景真实|客户状态、个人经历、数据和事实不能靠 AI 编。
语气适合|发给客户、领导、朋友、用户时，语气不能错位。
不过度承诺|不要为了显得专业而承诺自己不能保证的结果。
信息完整|关键时间、对象、下一步动作不能缺。
可发送|真正发出去前，自己能对这段话负责。
:::

AI 可以帮你生成更贴近场景的内容，但不能替你承担结果。

如果是发给客户、领导、公开平台，或者涉及商业承诺、财务、法律、医疗、隐私内容，最好再自己检查一遍。

## 我现在更相信的一件事

高质量 Prompt 不是一段很长、很复杂、看起来很专业的话。

更重要的是，它让 AI 明白五件事：你是谁，你要完成什么，背景是什么，有哪些限制，最终要输出成什么样。

普通人学 Prompt，不是为了背模板。

更有用的方式，是拆解一个好案例，看它哪些地方说清楚了，再把这些结构迁移到自己的任务里。

不要把 Prompt 当成万能句子。

更适合的方式，是把它当成一份清楚的任务说明。

## 今天把一句泛提示词改成高质量版本

今天先把一句自己常用的泛提示词改成高质量版本。

可以从这些句子里选一条：帮我写一段客户跟进话术；帮我写一篇小红书文案；帮我做一份学习计划；帮我总结这篇文章；帮我写一份周报。

任务目标不是马上得到最终内容，而是学会把泛提示词补充成更清楚的任务说明。

:::tutorial-flow
title: 改写练习卡
选一句泛提示词|从自己经常用的一句话开始，不要选太复杂的任务。
补充身份|说明你是谁，或者 AI 要扮演什么角色。
补充任务目标|说清这次到底要得到什么结果。
补充背景|交代当前状态、对象关系和为什么要做。
补充限制|说明不要什么语气、不要什么承诺、不要哪些内容。
规定输出格式|要求表格、清单、多个版本或步骤说明。
让 AI 检查缺口|让 AI 反过来指出这条 Prompt 还缺什么。
:::

做完以后，不要只看 AI 改写得顺不顺。你要看它有没有真的补上身份、任务、背景、限制和输出格式。

## 直接复制这段，帮你检查一条 Prompt

```text
我写了一条 Prompt，但不确定它够不够清楚。

我的 Prompt 是：

请你帮我检查这条 Prompt 是否完整，并按下面结构分析：

1. 任务是否清楚
2. 是否说明了使用场景
3. 是否补充了必要背景
4. 是否说明了限制条件
5. 是否规定了输出格式
6. 是否还缺少关键信息

请你再帮我做两件事：

1. 先指出这条 Prompt 的问题
2. 再帮我改成一个更清楚、更适合真实使用的版本

要求：

1. 不要写得太复杂
2. 不要堆专业术语
3. 保留我的原始目的
4. 如果信息不足，请先问我 3 个问题
5. 输出格式用表格
```

如果你已经补充了自己的场景，可以继续追问：

```text
请基于我的具体场景，再帮我改成一个可以直接使用的版本。
```

## 后面继续怎么练

如果你今天真的把一句泛提示词改成了更清楚的版本，就已经完成了一次有效练习。

以后看到别人的 Prompt，不要急着整段复制。

先看它有没有说清身份、任务、背景、限制和输出格式。

再把这些部分换成自己的真实场景。

这样你不是在背模板，而是在学会写自己的任务说明。
