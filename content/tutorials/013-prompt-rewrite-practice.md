## Prompt 不是一次写完的

很多人练 Prompt 时，会误以为高手是一上来就能写出完整提示词。

实际练习时更常见的是，第一版只是一句很普通的话。

帮我写一篇文章。

帮我做一份计划。

帮我写一段客户话术。

这些句子当然不够清楚，但它们不是没用。

它们更像第一版草稿。

下一步不是责怪自己不会写，也不是马上去找更长的模板，而是把这句话一轮一轮改清楚。

第一轮，先补任务和读者。第二轮，补背景、限制和不要什么。第三轮，再规定输出格式和检查方式。

这篇不讲更多理论，只练一件事：把一句模糊 Prompt，改成一条能真正开始使用的任务说明。

## 第一版很模糊，也可以继续改

很多人看到自己写出“帮我写一篇文章”这种句子，会觉得这条 Prompt 已经失败了。

其实不用这么想。

模糊 Prompt 最大的问题不是不能用，而是它让 AI 只能靠猜。

它不知道写给谁看，不知道文章用在哪里，不知道读者现在是什么水平，也不知道要避免什么语气。它更不知道最后要什么格式，写完后要不要检查。

所以第一版的作用，不是直接拿来执行，而是拿来继续加工。

:::tutorial-compare-table
title: 原始 Prompt vs 三轮改写
原始：帮我写一篇文章|只给了一个模糊方向，AI 需要猜读者、场景、语气和格式。
第一轮：补任务和读者|先说清写什么、写给谁看。
第二轮：补背景和限制|说明读者状态、语气边界和不要出现的方向。
第三轮：补格式和检查方式|规定先输出什么、怎么交付、写完后如何自检。
:::

你不需要一开始就把所有信息都写完。先让第一版变清楚一点，再继续往下改。

## 第一轮：先把任务和读者说清楚

我们先拿这句原始 Prompt 来练：

帮我写一篇文章。

第一轮不要急着补很多限制，只解决两个问题：这次到底要做什么，内容是写给谁看的。

可以先改成这样：

请帮我写一篇面向 AI 小白的文章，主题是“普通人如何开始使用 AI”。

这句话还不算完整，但已经比原来清楚很多。

原来的“写一篇文章”太宽。它可以是公众号文章、朋友圈长文、内部培训稿，也可以是课程开场稿。AI 不知道你要哪一种，就只能按常见套路写。

改成第一轮以后，AI 至少知道两个关键信息：主题是普通人如何开始使用 AI，读者是 AI 小白。

这一步不用想复杂。第一轮能把“做什么”和“给谁看”说清，就已经够了。

## 第二轮：补上背景、限制和不要什么

第一轮以后，任务和读者有了，但还不够。

同样是写给 AI 小白，有的人想写成工具介绍，有的人想写成学习路线，有的人想写成焦虑感很强的趋势文章。你不说，AI 还是会猜。

所以第二轮要继续补背景和限制。

可以这样改：

读者是没有技术基础的普通职场人，文章不要写得太技术，也不要制造焦虑，要用真实场景说明。

这一轮的作用，是减少 AI 乱猜。

你要告诉它：读者是什么状态，文章要用什么语气，哪些方向不要写，哪些感觉要避免，最好用什么方式讲清楚。

比如，不要写得太技术，不要制造焦虑，不要像营销文，多用真实场景说明。

这些限制不是为了束缚 AI，而是为了让结果更贴近你真正想要的方向。

## 第三轮：规定输出格式和检查方式

第三轮解决的是输出顺序、输出结构和怎么检查。

不要只是让 AI “写完”。你可以让它先给结构，再写正文，写完后自检，指出哪里空泛，给出修改建议。

继续补第三轮：

请先给我文章结构，再写正文。正文要有真实开头、3 个使用场景、一个可执行小任务。写完后请检查哪里像机器稿，并给出修改建议。

这样做有两个好处。

第一，你不用直接面对一整篇成品。AI 先给结构，你可以先判断方向对不对。

第二，你让 AI 参与检查，而不是只让它交一版内容。它不一定检查得完全准确，但可以帮你先发现一些明显问题，比如太空、太满、太像宣传语。

:::tutorial-flow
title: 三轮改写卡
任务和读者|先说清要做什么、写给谁看。
背景和限制|补充读者状态、使用场景、语气边界和不要出现的方向。
格式和检查|规定输出顺序、交付格式和自检方式。
:::

## 三轮之后，它会变成什么样？

三轮改完以后，最终 Prompt 可以变成这样：

请帮我写一篇面向 AI 小白的文章，主题是“普通人如何开始使用 AI”。

读者是没有技术基础的普通职场人。文章不要写得太技术，也不要制造焦虑，要用真实场景说明。

请先给我文章结构，再写正文。正文要有真实开头、3 个使用场景、一个可执行小任务。写完后请检查哪里像机器稿，并给出修改建议。

它不是变得很高级，而是变得更清楚。

这条 Prompt 至少说明了任务、读者、背景、限制、输出顺序和检查方式。

这就是普通人练 Prompt 最应该先掌握的能力：不是把句子写得更神秘，而是把任务说明写得更具体。

## 让 AI 反过来帮你改 Prompt

很多时候，你不知道自己的 Prompt 缺什么。

这时候可以先让 AI 检查 Prompt，而不是直接执行任务。

你可以这样问：

请先不要执行我的 Prompt。

请你先检查它是否清楚，包括：

1. 任务是否明确
2. 读者是否清楚
3. 背景是否足够
4. 限制是否明确
5. 输出格式是否清楚
6. 是否需要先问我补充信息

如果不完整，请先问我 3 个问题，再帮我改写。

让 AI 提问，不是你水平不够。

这是把模糊任务变清楚的一部分。

:::tutorial-flow
title: AI 反向检查 Prompt 卡
任务|这次到底要完成什么，是否说清楚。
读者|结果是给谁看的，是否有具体对象。
背景|为什么要做这件事，当前处在什么状态。
限制|哪些语气、方向、内容不要出现。
输出格式|要表格、清单、多个版本，还是步骤说明。
需要追问|信息不足时，先问问题再继续。
:::

你不确定哪里缺，就让 AI 先帮你找缺口。这样比反复让它“重新写一遍”更有效。

## 哪些结果不能只看清不清楚

Prompt 改得更清楚，可以减少 AI 乱猜，但不代表输出一定正确、一定自然、一定适合发布。

你仍然需要判断：任务是否真实，背景是否准确，限制是否足够，输出是否空泛，结果是否适合自己的场景。

:::tutorial-flow
title: 结果判断卡
任务真实|这件事确实是你要做的，不是为了练习而虚构一个假任务。
背景准确|读者、对象、经历、数据和使用场景不能靠 AI 编。
限制足够|该避免的语气、承诺、夸张表达要提前说清。
输出不空泛|结果不能只有正确废话，要能继续使用或修改。
适合场景|发给客户、领导、朋友或公开平台前，自己能负责。
:::

Prompt 改写是为了让 AI 少猜一点，不是为了让人完全不用判断。

尤其涉及客户沟通、公开发布、商业承诺、医疗健康、法律责任、财务收益和隐私信息时，AI 的输出只能作为初稿，最终判断仍然要由人负责。

## 我现在更相信的一件事

普通人练 Prompt，不是从背模板开始，而是从修改自己的模糊表达开始。

一条好 Prompt，往往不是一次写出来的。

它通常经历几轮改写。

第一轮：把任务和读者说清楚。

第二轮：把背景、限制和不想要的内容补上。

第三轮：把输出格式和检查方式定下来。

不要把 Prompt 当成一次性写完的标准答案。

更适合的理解是：它是一份可以反复修改的任务说明。

## 今天改一条自己常用的 Prompt

今天先改一条自己常用的 Prompt。

可以从这些原始句子里选一条：

1. 帮我写一篇文章
2. 帮我做一份计划
3. 帮我写一段客户话术
4. 帮我总结一份资料
5. 帮我写一条朋友圈

任务目标不是得到最终内容，而是得到一条更清楚、可执行、能继续追问的 Prompt。

:::tutorial-flow
title: Prompt 改写任务卡
原始 Prompt|先写下自己真的会用的一句话，不要选太复杂的任务。
第一轮|补任务和读者，说清做什么、给谁看。
第二轮|补背景、限制和不要什么，减少 AI 乱猜。
第三轮|补输出格式和检查方式，让结果更容易判断。
最终版本|整理成一条能开始使用、还能继续追问的 Prompt。
:::

做完以后，不要只看这条 Prompt 变长了没有。你要看它有没有真的更清楚：任务是否明确，读者是否具体，限制是否能帮你避开不想要的方向，输出方式是否方便检查。

## 直接复制这段，让 AI 帮你改 Prompt

```text
我写了一条 Prompt，但它可能太模糊了。

我的原始 Prompt 是：

请你先不要执行它，而是帮我做一次改写练习。

请按下面步骤处理：

第一步：
判断这条 Prompt 缺少哪些信息。

第二步：
帮我补成三个版本：

1. 第一轮改写：补清楚任务和读者
2. 第二轮改写：补清楚背景、限制和不要什么
3. 第三轮改写：补清楚输出格式和检查方式

第三步：
给出最终可使用版本。

第四步：
告诉我这条 Prompt 以后还可以怎么继续改。

要求：

1. 不要使用专业术语
2. 不要直接替我完成任务
3. 重点是帮我学会如何改 Prompt
4. 如果信息不足，请先问我 3 个问题
5. 输出格式用表格
```

如果你已经有具体场景，可以继续追问：

```text
请基于我的真实使用场景，把第三轮版本再改得更具体。
```

## 后面继续怎么练

如果你今天真的把一句“帮我写一下”改成了三轮版本，就已经完成了一次有效练习。

Prompt 能力不是靠收藏变强的，也不是靠背模板变强的。

它是在一次次改写里变清楚的。

以后写任何 Prompt，都可以先问自己三件事。

任务和读者说清了吗？

背景和限制补上了吗？

输出格式和检查方式说明了吗？

当你能这样检查自己的提问时，就不是在找现成答案，而是在练一种可以迁移到很多任务里的能力。
