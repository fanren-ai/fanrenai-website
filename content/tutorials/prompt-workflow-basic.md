# Prompt和工作流：从一句话提问到稳定产出

多数人用AI效率不稳定，不是因为工具不够强，而是因为提问方式太随意。Prompt的价值在于把你的真实需求翻译成AI能执行的任务说明。

## 一个基础Prompt结构

可以先用这个结构：

- 角色：你希望AI以什么身份处理任务
- 目标：这次要完成什么
- 背景：已有信息和使用场景
- 要求：风格、边界、重点
- 格式：你希望它怎么输出

## 把Prompt变成工作流

一套简单的写作工作流可以这样设计：

1. 让AI根据主题生成选题角度
2. 选择一个角度后生成提纲
3. 根据提纲写初稿
4. 检查逻辑漏洞和表达冗余
5. 按目标读者重写标题和开头

## 为什么要沉淀模板

当你发现某个Prompt能稳定产出结果，就应该把它保存下来。未来你只需要替换主题、场景和素材，就能快速复用。

## 实战建议

不要追求一次写出完美Prompt。先让AI产出，再用“哪里不对”“为什么不够好”“按这个方向改”持续校准，这才是普通人最容易掌握的路线。
